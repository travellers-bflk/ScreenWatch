# 安全策略

## 报告安全漏洞

我们非常重视 ScreenWatch 的安全问题。如果你发现安全漏洞，请按照以下步骤私下报告：

1. **GitHub Security Advisories**：请在 GitHub 仓库的 Security 标签页中创建私密 Security Advisory（推荐方式）。
2. **邮件报告**：你也可以发送邮件至 security@screenwatch.example（请勿在公开 issue 中讨论安全漏洞细节）。

请提供以下信息以便我们快速处理：

- 漏洞的详细描述
- 复现步骤（如有）
- 影响范围评估
- 建议的修复方案（如有）

## 响应时间

| 步骤 | 时间承诺 |
|------|---------|
| 确认收到报告 | 48 小时内 |
| 初步评估 | 5 个工作日内 |
| 修复方案反馈 | 14 天内 |
| 修复发布 | 视严重程度而定，通常 30 天内 |

## 支持的版本

| 版本 | 是否支持安全更新 |
|------|-----------------|
| 最新发布版 | ✅ 是 |
| 之前的次要版本 | ⚠️ 仅限严重漏洞 |
| 更早版本 | ❌ 否 |

## 隐私说明

ScreenWatch 是一款**完全本地化**的屏幕使用时间统计工具：

- **无网络上报**：本软件不会将任何数据发送到网络服务器
- **无遥测**：不包含任何遥测或用户行为追踪代码
- **无第三方分析 SDK**：不集成任何第三方数据收集组件
- **数据本地存储**：所有统计数据仅保存在用户本地设备的 SQLite 数据库中

因此，ScreenWatch 的安全漏洞主要影响范围是**本地数据安全**，例如：

- 本地数据库文件被未授权读取
- 配置文件中的敏感信息泄露
- 本地权限提升

若发现涉及数据外传或网络上报行为的漏洞，这属于**最高优先级**，我们将立即处理。

## 安全措施

- 所有用户数据文件在 `.gitignore` 中被严格排除，防止误提交到仓库
- CI 流水线集成 CodeQL 自动安全扫描
- 依赖项定期更新以修复已知漏洞
